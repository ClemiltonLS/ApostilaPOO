Programação Procedural

Paradigma de programação baseado no conceito de chamadas a procedimentos. Procedimentos também podem ser chamados de rotinas, subrotinas, métodos ou funções. Simplesmente contêm um conjunto de passos computacionais a serem executados de forma sequêncial. Um procedimento pode ser executado a qualquer momento pela aplicação principal, por outros métodos ou por ele mesmo.

Características:

  • Habilidade de reutilizar o mesmo código em diferentes lugares no programa sem copiá-lo.

  • Mais fácil de organizar o fluxo do programa.

  • Habilidade de ser fortemente modulada.

Programação Orientada a Objetos

P.O.O é um paradigma de programação que usa "objetos" compostos de atributos e métodos, juntamente com suas interações para projetar programas de computador

Organiza o mundo real como uma coleção de objetos que incorporam estruturas de dados e um conjunto de operações que manipulam estes dados e trocam informações entre si.

Características:

  • Melhor habilidade de reutilizar códigos.

  • Organização do fluxo do programa é mais complexa.

  • Aprendizagem mais complexa.

Vantagens

Vantagens Diretas – maior facilidade para reutilização de código e por conseqüência do projeto – possibilidade do desenvolvedor trabalhar em um nível mais elevado de abstração – utilização de um único padrão conceitual durante todo o processo de criação de software – maior adequação à arquitetura cliente/servidor – maior facilidade de comunicação com os usuários e com outros profissionais de informática.

Vantagens Reais – ciclo de vida mais longo para os sistemas – desenvolvimento acelerado de sistemas – possibilidade de se construir sistema muito mais complexos, pela incorporação de funções prontas – menor custo para desenvolvimento e manutenção de sistemas.

Desvantagens

Complexidade no aprendizado para desenvolvedores de linguagens estruturadas;

Maior uso de memória (heap), por exemplo para aplicações móveis em JavaME;

Maior esforço na modelagem de um sistema OO do que estruturado (porém menor esforço de codificação, sendo uma vantagen);

Funcionalidades limitadas por interface, quando estas estão incompletas (problemas na modelagem);

Dependência de funcionalidades já implementadas em superclasses no caso da herança, implementações espalhadas em classes diferentes;

Classes

Conjunto de características e ações comuns a um grupo de indivíduos;

Molde para descrição das características e ações de um grupo de objetos;

Arquivo contendo o código fonte necessário para gerar instâncias de objetos, composições ou heranças;

Classes sempre são criadas na ideia de pluralidade, onde não se justifica criar uma classe que possui características e ações pertencentes somente a um indivíduo;

As características também podem ser chamadas de propriedades;

No código fonte, as características são chamadas de atributos, e utilizam o mesmo padrão das variáveis da linguagem utilizada;

As ações também podem ser chamados comportamentos

No código fonte, as ações são chamadas de métodos, e utilizam o mesmo padrão para procedimentos/funções ou métodos da linguagem utilizada;

O código fonte de uma classe deve conter somente os atributos e métodos referêntes à mesma. Exclusivamente em C, os arquivos de classe devem receber a extensão .cpp e .h

Objetos

Indivíduo com as características e ações de uma classe;

Representante de uma classe;

Membro de uma classe;

Instancia

Execução de uma classe;

Objeto ativo;

Objeto de uma classe armazenado dentro de espaço em memória para utilização na aplicação;

Empacotamento

Pacotes são unidades de organização para as classes de seu projeto.

Também são amplamente utilizados para organização de bibliotecas padrão da linguagem ou de terceiros.

A definição do pacote deve acontecer, obrigatoriamente na primeira linha no código fonte das classes.

Importação

Recurso que permite utilizarmos de código de classes que estejam em diferentes pacotes.

Também pode ser chamado de include

A importação de classes ou pacotes inteiros deve acontecer após a definição do pacote e antes da definição da classe no código fonte

Empacotamento na prática
C++ - No arquivo Classe.h
#ifndef NOMEPACOTE_NOMECLASSE_H_
#define NOMEPACOTE_NOMECLASSE_H_
Java
package NomePacote;
Python
Definição a variável de Ambiente PYTHONPATH 
ou 
coloca-se o arquivo da classe dentro de um diretório 
que tenha o arquivo __init__.py
Importação na prática
C++ 
#include "pacote/Classe.h"
Java
import pacote.Classe
Python
Se a variável PYTHONPATH for editada
import pacote.Classe
Senão
from Pacote.Classe import Classe
Composição | Associação

É considerado Composição quando uma classe possui um atributo que seja uma instância de outra classe.

É considerado Associação quando uma classe é instânciada como atributo de outra classe.

Tanto Composição quanto Associação referem-se ao mesmo tipo de técnica, mudando somente o ponto de perspectiva.

É uma técnica para reutilização de código, que visa diminuir o tempo de desenvolvimento de aplicações em que várias classes possuem o mesmo atributo, porém não se aplica uma Herança.
